프로그래머스 1단계 : 예산

코딩테스트 연습 > Summer/Winter Coding(~2018) > 예산

💩 문제 설명

S사에서는 각 부서에 필요한 물품을 지원해 주기 위해 부서별로 물품을 구매하는데 필요한 금액을 조사했습니다. 그러나, 전체 예산이 정해져 있기 때문에 모든 부서의 물품을 구매해 줄 수는 없습니다. 그래서 최대한 많은 부서의 물품을 구매해 줄 수 있도록 하려고 합니다.
물품을 구매해 줄 때는 각 부서가 신청한 금액만큼을 모두 지원해 줘야 합니다. 예를 들어 1,000원을 신청한 부서에는 정확히 1,000원을 지원해야 하며, 1,000원보다 적은 금액을 지원해 줄 수는 없습니다.
부서별로 신청한 금액이 들어있는 배열 d와 예산 budget이 매개변수로 주어질 때, 최대 몇 개의 부서에 물품을 지원할 수 있는지 return 하도록 solution 함수를 완성해주세요.

💩 제한 조건

☝️ d는 부서별로 신청한 금액이 들어있는 배열이며, 길이(전체 부서의 개수)는 1 이상 100 이하입니다.
✌ d의 각 원소는 부서별로 신청한 금액을 나타내며, 부서별 신청 금액은 1 이상 100,000 이하의 자연수입니다.
👌 budget은 예산을 나타내며, 1 이상 10,000,000 이하의 자연수입니다.

💩 문제 풀이

function solution(d, budget) {
    d.sort((a, b) => a - b);
    for(i=0; i<d.length; i++){
        if(budget < d[i]) { break; }
        budget -= d[i];
    }
    return i;
}

신청 금액을 작은 순서로 먼저 정렬해주고, for문을 사용하여 금액을 하나씩 예산에서 빼 주었습니다.

💩 다른 풀이

function solution(d, budget) {
    d.sort((a, b) => a - b);
    while (d.reduce((a, b) => (a + b), 0) > budget) d.pop();
    return d.length;
}

다른 풀이들이 다 휘황찬란해서 제가 이해할 수 있는 풀이가 이것 밖에 없네요... 예산을 넘지 않는 한까지 d 내의 모든 원소들을 더해주고, 예산을 넘으면 원소를 제거하여 최종적인 배열의 길이를 반환하였습니다.
